# Squad Health Check

Squad Health Check — мероприятие, помогающее команде найти решение самой актуальной проблемы.

Фактически, это альтернативный формат командной ретроспективы, на которой:
- Команда оценивает состояние дел в разных аспектах своей деятельности
- Выявляет наиболее проблемные направления
- Собирает связанные с ними проблемы и препятствия (анализирует причины неидеального состояния)
- Выбирает самые важные из них
- Ищет способы их решения

После встречи остаются артефакты:
- **Результат оценки.** Его можно использовать в следующий раз для сравнения с предыдущими результатами. Это помогает оценить эффективность выбранных решений
- **Приоритезированный список текущих проблем.** Решив самую важную, можно перейти к следующей
- **План решения ключевой проблемы**

При разработке методики проведения мы вдохновлялись примером из [Spotify](https://engineering.atspotify.com/2014/09/squad-health-check-model/).

## Зачем нужно

Squad Health Check помогает команде систематически улучшать свою работу. 
Регулярные встречи позволяют сравнить текущее состояние с предыдущими результатами и оценить динамику изменений.

Анализ данных по всем командам подразделения помогает выявить общие проблемы и разработать универсальные решения.

## Как проводить

Мероприятие проводится в формате командной ретроспективы. Основное отличие — этап сбора данных:
- Команда оценивает состояние дел по стандартным критериям, используя шкалу: **хорошо/средне/плохо**
- Затем определяется тренд: **улучшается/не меняется/ухудшается**

Дальнейшее обсуждение фокусируется на самых слабых местах.

Для помощи командам доступны:
- [шаблон в Miro](https://miro.com/app/board/uXjVJmLLLns=/?share_link_id=44122416350). (Введите: Health_Check_1)
- [рекомендации по проведению](shc_how_to_external.md).

**Анонимность.** Оценка может проводиться анонимно. Формат остальных этапов команда выбирает самостоятельно. 
Если участники не готовы к открытому обсуждению, рекомендуется сохранить анонимность для всех этапов.

**Рекомендация:** первый раз проведите мероприятие полностью анонимно.

**Длительность.** От 1 часа. Точное время зависит от размера команды, опыта фасилитатора и количества проблем. 
Для первого раза выделите 1.5–2 часа.

**Регулярность.** Проводите на постоянной основе. Оптимальная периодичность: 
- Не реже 1 раза в год
- Не чаще 1 раза в квартал
- Рекомендуемый интервал — раз в полгода

**Отличия при первом и последующих проведениях:**
- Первый раз: изучение критериев и шкалы оценки, определение тренда по ощущениям команды
- Последующие разы: добавление сравнения с предыдущими результатами

## Что оцениваем

Стандартный список критериев, подходящий большинству команд. Критерии сгруппированы для удобства:

**Задачи и цели** (понимание и влияние команды):
- Понятность миссии и целей
- Участие в принятии решений

**Результат работы команды** (эффективность):
- Качество
- Скорость
- Удовлетворённость заказчиков

**Внутренние факторы результата:**
- Командная работа
- Процесс разработки
- Экспертиза
- Дежурство  
- Кодовая база

**Внешние факторы результата:**
- Инструменты
- Взаимодействие со смежными командами

**Личные аспекты:**
- Удовольствие от работы  
- Обучение и развитие

Подробное описание каждого критерия доступно в шаблоне Miro.
